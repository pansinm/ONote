## 系统概述

ONote 是一个基于 Electron 的跨平台桌面笔记应用，采用 Monorepo 架构。应用支持 Markdown 编辑、实时预览、AI 集成、多数据源等丰富功能。

### 技术栈

- **桌面框架**: Electron 35.7.5
- **UI 框架**: React 18.2.0
- **编程语言**: TypeScript 5.9.3
- **状态管理**: MobX 6.4.1
- **编辑器**: Monaco Editor 0.45.0
- **UI 组件库**: @fluentui/react-components 9.49.2
- **构建工具**: Webpack 5, Yarn Workspaces
- **包管理**: Yarn Workspaces

### 核心特性

- 📝 Markdown 实时编辑与预览
- 🌳 多数据源支持（本地、SSH、Git）
- 🔌 插件化架构
- 🤖 AI 集成（OpenAI API）
- 📊 图表支持（Mermaid、PlantUML、Graphviz）
- 🎨 数学公式渲染（KaTeX）
- 🖥️ 多窗口支持（主窗口、预览窗口）
- 🌐 WebDAV 服务器

## 架构设计

### 整体架构

ONote 采用经典的 Electron 多进程架构：

```
┌─────────────────────────────────────────────────────┐
│                   ONote Application                  │
├─────────────────────────────────────────────────────┤
│                                                       │
│  ┌──────────────────────┐      ┌─────────────────┐ │
│  │   Main Process       │◄────►│ Renderer Process│ │
│  │  (Node.js Context)   │ IPC  │  (Web Context)  │ │
│  │                      │      │                 │ │
│  │  - Window Management │      │  - React UI     │ │
│  │  - File System       │      │  - Monaco Editor│ │
│  │  - IPC Server        │      │  - Previewer    │ │
│  │  - Plugin Manager    │      │  - MobX Stores  │ │
│  │  - Data Sources      │      │  - Components   │ │
│  │  - WebDAV Server     │      │                 │ │
│  └──────────────────────┘      └─────────────────┘ │
│                                           │          │
│                                    ┌──────┴──────┐  │
│                                    │ Shared Code │  │
│                                    │  (Types,    │  │
│                                    │   Utils)    │  │
│                                    └─────────────┘  │
└─────────────────────────────────────────────────────┘
```

### Monorepo 结构

项目采用 Yarn Workspaces 管理的 Monorepo 架构：

```
ONote/
├── packages/
│   ├── electron/      # 主进程代码
│   ├── renderer/      # 渲染进程代码
│   └── shared/        # 共享代码
├── buildResources/    # 构建资源
├── docs/             # 文档
└── [配置文件]
```

## 核心模块

### 1. 主进程 (Main Process)

主进程是应用的入口，负责创建和管理应用窗口、系统级操作。

**主要职责：**
- 应用生命周期管理
- 窗口创建和管理
- 系统托盘
- 原生菜单
- 文件系统访问
- IPC 通信服务器

**入口文件**: [packages/electron/src/index.ts](../packages/electron/src/index.ts)

**关键代码：**
```typescript
// 应用启动流程
app.whenReady()
  .then(() => startIpcServer())        // 启动 IPC 服务器
  .then(() => restoreOrCreateWindow()) // 创建主窗口
  .then(() => pluginManager.loadAll()) // 加载插件
```

### 2. 渲染进程 (Renderer Process)

渲染进程运行用户界面，基于 React 构建。

**主要职责：**
- UI 渲染和交互
- Monaco Editor 集成
- Markdown 预览
- 用户输入处理
- 状态管理

**入口文件**: [packages/renderer/src/main/index.tsx](../packages/renderer/src/main/index.tsx)

**组件层级：**
```
App
├── Sidebar         # 侧边栏（文件树）
├── Page            # 主页面
│   ├── Editor      # 编辑器
│   └── Previewer   # 预览器
└── EventBus        # 事件总线
```

### 3. IPC 通信 (Inter-Process Communication)

主进程和渲染进程通过 IPC 进行通信，使用命名空间组织。

**IPC 服务器**: [packages/electron/src/ipc-server/](../packages/electron/src/ipc-server/)

**命名空间：**
```typescript
enum IPCNamespaces {
  DataSource = 'DataSource',      // 数据源操作
  Setting = 'Setting',            // 设置管理
  DevelopTools = 'DevelopTools',  // 开发工具
  PluginManager = 'PluginManager', // 插件管理
  Cron = 'Cron',                  // 定时任务
  Typst = 'Typst',                // Typst 渲染
}
```

**通信流程：**
```
Renderer Process          Main Process
     │                        │
     │  ipcRenderer.invoke    │
     ├──────────────────────►│
     │                        │  Handler.process()
     │                        │
     │  ◄─────────────────────┤
     │     result/error       │
```

### 4. 数据源系统 (Data Sources)

支持多种数据源，抽象接口设计。

**接口定义**: [packages/electron/src/dataSource/DataSource.ts](../packages/electron/src/dataSource/DataSource.ts)

**支持的数据源：**
- **LocalDataSource**: 本地文件系统
- **SSHDataSource**: 远程 SSH 服务器
- **GiteeDataSource**: Gitee 仓库

**主要方法：**
```typescript
interface DataSource {
  list(path: string): Promise<FileEntry[]>
  read(path: string): Promise<string>
  write(path: string, content: string): Promise<void>
  watch(path: string, callback: Function): Promise<Closeable>
  // ...
}
```

### 5. 插件系统 (Plugin System)

支持动态加载插件扩展功能。

**插件管理器**: [packages/electron/src/plugin/PluginManager.ts](../packages/electron/src/plugin/PluginManager.ts)

**插件目录**: [packages/electron/src/plugin/plugins/](../packages/electron/src/plugin/plugins/)

## 通信机制

### IPC 通信架构

ONote 使用自定义的 IPC 服务器实现主进程和渲染进程的通信。

**IPC 服务器基类**: [packages/electron/src/ipc-server/IPCServer.ts](../packages/electron/src/ipc-server/IPCServer.ts)

**关键特性：**
- 命名空间隔离
- 自动错误处理
- 类型安全
- 事件委托

**使用示例：**

渲染进程调用：
```typescript
const result = await ipcRenderer.invoke('DataSource.list', '/path/to/dir');
```

主进程处理：
```typescript
class DataSourceHandler {
  async list(event: IpcInvokeEvent, path: string) {
    return await dataSource.list(path);
  }
}
```

### 事件委托

自动将数据源事件转发到渲染进程：

```typescript
delegateEvent(
  dataSource,
  EventNames.FileContentChanged,
  IPCNamespaces.DataSource,
  () => webContents.fromFrame(getMainFrame()!)!,
);
```

## 状态管理

### MobX Store 架构

使用 MobX 进行响应式状态管理，所有 Store 位于 [packages/renderer/src/main/stores/](../packages/renderer/src/main/stores/)。

**核心 Store：**

#### 1. ActivationStore
管理应用激活状态和 UI 可见性。

```typescript
class ActivationStore {
  hideSidebar = false;
  currentFileId: string | null = null;

  activateFile(fileId: string) { ... }
  toggleSidebar() { ... }
}
```

#### 2. FileListStore
管理文件列表状态。

```typescript
class FileListStore {
  files: ObservableArray<FileEntry> = [];
  selectedFiles: Set<string> = new Set();

  loadFiles(path: string) { ... }
  selectFile(fileId: string) { ... }
}
```

#### 3. FileStore
管理当前打开文件的状态。

```typescript
class FileStore {
  content: string = '';
  isDirty: boolean = false;

  load(fileId: string) { ... }
  save() { ... }
}
```

#### 4. LayoutStore
管理布局状态（编辑器/预览器布局）。

```typescript
class LayoutStore {
  layout: 'editor' | 'preview' | 'split' = 'split';

  setLayout(layout: LayoutType) { ... }
}
```

#### 5. SettingStore
管理应用设置。

```typescript
class SettingStore {
  theme: string = 'light';
  fontSize: number = 14;

  updateSetting(key: string, value: any) { ... }
}
```

#### 6. TodoStore
管理待办事项。

```typescript
class TodoStore {
  todos: ObservableArray<TodoItem> = [];

  addTodo(text: string) { ... }
  toggleTodo(id: string) { ... }
}
```

**Store 入口**: [packages/renderer/src/main/stores/index.ts](../packages/renderer/src/main/stores/index.ts)

### 状态更新流程

```
User Action
     │
     ▼
Component Event Handler
     │
     ▼
MobX Store Action (@action)
     │
     ▼
Observable State Update
     │
     ▼
Auto React Re-render
```

## 数据流

### 典型操作流程

#### 1. 打开文件流程

```
User: Click File in Sidebar
     │
     ▼
Sidebar Component
     │
     ▼
FileListStore.selectFile(fileId)
     │
     ▼
FileStore.load(fileId)
     │
     ├──────────────────► IPC: DataSource.read(path)
     │                        │
     │                        ▼
     │                   Main Process
     │                        │
     │                        ▼
     │                   DataSource.read()
     │                        │
     ◄───────────────────────┘
     │
     ▼
Update FileStore.content
     │
     ▼
Monaco Editor Update
```

#### 2. 保存文件流程

```
User: Ctrl+S
     │
     ▼
Editor Component
     │
     ▼
FileStore.save()
     │
     ├──────────────────► IPC: DataSource.write(path, content)
     │                        │
     │                        ▼
     │                   Main Process
     │                        │
     │                        ▼
     │                   DataSource.write()
     │                        │
     ◄───────────────────────┘
     │
     ▼
Update FileStore.isDirty = false
```

#### 3. Markdown 预览流程

```
Monaco Editor Content Change
     │
     ▼
Editor Component
     │
     ▼
Previewer Component
     │
     ├──────────────► Markdown AST (unified-mdast)
     │                │
     │                ▼
     │            Handler Pipeline
     │                │
     │                ├─► Code Handler (prismjs)
     │                ├─► Math Handler (katex)
     │                ├─► Mermaid Handler
     │                └─► ...
     │                │
```

## 构建系统

### Yarn Workspaces

使用 Yarn Workspaces 管理 Monorepo 的依赖和构建流程。

**配置文件**: [package.json](../../package.json)

**主要特性：**
- 统一依赖管理：所有 workspace 共享 node_modules
- 并行构建：`yarn workspaces run build` 并行构建所有包
- 增量安装：只安装必要的依赖

**常用命令：**
- `yarn install` - 安装所有 workspace 依赖
- `yarn workspaces run build` - 构建所有包
- `yarn workspace <name> <command>` - 运行特定 workspace 命令

### Webpack 配置

渲染进程使用 Webpack 打包。

**配置文件**: [packages/renderer/webpack.config.js](../packages/renderer/webpack.config.js)

**关键特性：**
- TypeScript 加载
- SCSS 支持
- Monaco Editor Worker 配置
- 开发服务器热更新
