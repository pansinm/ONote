# ONote 功能清单（详细版）

> 从用户视角全面了解 ONote 的所有功能

## 项目概述

ONote 是一款基于 **Electron + React + TypeScript** 构建的跨平台桌面笔记应用，专为 Markdown 编辑、知识管理和 AI 辅助写作设计。

**核心特性：**
- 📝 强大的 Markdown 编辑器（基于 Monaco Editor）
- 👁️ 实时预览与双向同步
- 🤖 内置 AI 对话助手（LLMBox）
- 📊 多数据源支持（本地、SSH、Gitee）
- ✅ 待办事项管理
- 🔌 可扩展插件系统
- 🔄 WebDAV 多设备同步

## 1. Markdown 编辑器功能

### 1.1 基础编辑功能

#### 语法高亮
- ✅ **Markdown 语法高亮** - 完整支持标准 Markdown 语法
- ✅ **代码块语法高亮** - 支持数百种编程语言（通过 Prism.js）
- ✅ **代码折叠** - 可折叠代码块和章节
- ✅ **Vim 模式** - 支持 Vim 编辑模式
- ✅ **多光标编辑** - 支持多光标同时编辑

#### 快捷操作
- ⌨️ **保存文件** - `Ctrl/Cmd + S`
- ⌨️ **切换任务状态** - `Alt + D`
- ⌨️ **粘贴** - `Ctrl/Cmd + V`（增强粘贴，支持图片）

### 1.2 智能补全系统

#### 快速插入补全（@触发）
**触发方式：** 输入 `@` 字符

**可用选项：**
- `@riqi` - 插入当前日期（格式：`YYYY年MM月DD日`）
- `@shijian` - 插入当前时间（格式：`HH:mm:ss`）

#### Emoji 补全（:触发）
**触发方式：** 输入 `:` 字符

**功能特点：**
- 基于 `node-emoji` 库，支持所有标准 Emoji
- 输入 Emoji 名称（如 `:smile:`）自动补全
- 自动生成 `:emoji_name:` 格式
- 代码块内不生效

**示例：**
```markdown
:smile → 😊
:heart → ❤️
:thumbsup → 👍
```

#### 代码块语言补全（```触发）
**触发方式：** 输入三个反引号 `` ` ``

**功能特点：**
- 支持 Monaco Editor 所有编程语言
- 自动生成完整代码块结构
- 光标自动定位到代码内容区

**示例：**
````
```javascript
// 光标位置
```
````

#### 路径补全（/触发）
**触发方式：** 在链接或图片路径中输入 `/`

**支持的格式：**
- Markdown 链接：`[text](path)`
- Markdown 图片：`![alt](path)`

**功能特点：**
- 自动解析当前文件所在目录
- 支持相对路径和绝对路径
- 文件夹补全后自动添加 `/`
- 智能过滤隐藏文件（以 `.` 开头）
- 按类型排序（文件夹优先，文件在后）

### 1.3 特殊编辑模式

#### 思维导图模式
**触发条件：** 代码块语言为 `mindmap` 时自动启用

**特殊快捷键：**
- `Enter` - 自动保持当前缩进层级
- `Tab` - 增加层级（转换为 `++` 或 `**`）
- `Shift + Tab` - 减少层级（移除标记）

#### 智能粘贴
**功能特点：**
- 粘贴图片时自动保存到 `assets/` 目录
- 自动生成 Markdown 图片语法 `![alt](path)`
- 支持 HTML 图片元素
- 普通文本正常粘贴

### 1.4 编辑器命令

#### 日期插入命令
- **命令 ID：** `onote.command.insertDate`
- **功能：** 在当前位置插入当前日期

#### 时间插入命令
- **命令 ID：** `onote.command.insertTime`
- **功能：** 在当前位置插入当前时间

#### 任务列表切换
- **命令 ID：** `onote.action.toggleTaskListAction`
- **快捷键：** `Alt + D`
- **功能：** 切换任务完成状态
  - `[ ]` → `[x]`（未完成 → 完成）
  - `[x]` → `[ ]`（完成 → 未完成）

### 1.5 编辑器配置

#### 可自定义项
- ✅ 主题设置（明暗主题）
- ✅ 字体设置（字体族、字号）
- ✅ 编辑器行为配置
- ✅ 快捷键自定义
- ✅ 布局设置（侧边栏宽度、文件列表宽度）

## 4. 多数据源支持

### 4.1 本地文件系统

**数据源 ID：** `local`

**特点：**
- 无需认证
- 直接访问本地文件
- URI 格式：`file:///absolute/path/to/file.md`
- 自动创建父目录
- 使用文件修改时间作为版本号

### 4.2 SSH 远程服务器

**数据源 ID：** `ssh`

**认证方式：**
- 用户名 + 密码
- 用户名 + 私钥

**特点：**
- 基于 SSH2/SFTP 协议
- 远程文件读写
- 自动下载缓存到本地临时目录
- 使用 MD5 哈希标识会话
- 自动重连机制

**配置示例：**
```typescript
{
  host: 'example.com',
  port: 22,
  username: 'user',
  password: 'password'  // 或使用 privateKey
}
```

### 4.3 Gitee 云端仓库

**数据源 ID：** `gitee`

**认证方式：** Personal Access Token

**特点：**
- 使用 Gitee API v5
- 仓库列表获取
- Git 操作（提交、删除）
- 文件内容 Base64 编码
- SHA 跟踪用于更新和删除

**URI 格式：** `file:///{namespace}/{repo}/{path}`

### 4.4 统一操作接口

所有数据源支持相同的操作：
- ✅ 文件读写（二进制和文本）
- ✅ 目录操作（创建、列出、删除）
- ✅ 文件重命名和移动
- ✅ 递归文件搜索
- ✅ 版本号获取
- ✅ 文件缓存到本地

## 7. 插件系统

### 7.1 插件接口规范

```typescript
interface IPlugin {
  name: string;                    // 插件名称（唯一标识）
  mainJs?: string;                 // 主进程 JS 路径
  previewerJs?: string;           // 预览窗口 JS 路径
  backendJs?: string;             // 后台 JS 路径
  author: string;                  // 作者
  version: string;                // 版本号
  title: string;                  // 显示标题
  description?: string;            // 描述
  installDir: string;              // 安装目录
  homepage?: string;               // 主页
  downloadUrl: string;            // 下载地址
  logo?: string;                  // logo 路径
  hasUpdate?: boolean;             // 是否有更新
  state?: 'installed' | 'uninstalled'; // 安装状态
}
```

### 7.2 插件生命周期

#### 安装流程
1. 支持 HTTP/HTTPS、本地文件、file URL 三种安装源
2. 自动下载并解压 tgz 格式插件包
3. 解析 package.json 获取插件信息
4. 自动卸载已存在的同名插件

#### 加载机制
- **主进程** - 加载 backendJs，调用 setup 函数
- **渲染进程** - 动态注入 mainJs 和 previewerJs
- 通过 `window.__plugins` 暴露插件列表
- 提供插件环境就绪事件 (`onote:ready`)

#### 卸载流程
- 清理插件资源
- 清理 require 缓存
- 删除插件目录

### 7.3 插件功能支持

#### 前端脚本注入
- ✅ 主窗口脚本注入（mainJs）
- ✅ 预览窗口脚本注入（previewerJs）
- ✅ 自动处理框架加载顺序

#### 后端 API 扩展
- ✅ 注册 IPC 处理器
- ✅ 访问主进程所有服务（数据源、设置等）

### 7.4 示例插件

#### 内置示例
- ✅ 图片标注插件
- ✅ Drawio 插件
