## 第一课：理解项目结构

### 1.1 什么是 ONote？

ONote 是一个基于 **Electron** 的桌面笔记应用。简单来说：

- **Electron** = 用 Web 技术构建桌面应用的框架
- **Monorepo** = 在一个仓库中管理多个相关项目
- **主进程** = Electron 的后端，运行 Node.js
- **渲染进程** = Electron 的前端，运行 Chromium

### 1.2 目录结构概览

打开项目文件夹，你会看到：

```
ONote/
├── packages/           # 包含三个子项目
│   ├── electron/      # 主进程（后端）
│   ├── renderer/      # 渲染进程（前端）
│   └── shared/        # 共享代码
├── package.json       # 项目配置和依赖（Yarn Workspaces）
└── tsconfig.json      # TypeScript 配置
```

### 1.3 三个核心包

#### packages/electron - 主进程

**作用：** 应用的"大脑"，负责系统级操作

**类比：** 像餐厅的后厨

**主要功能：**
- 创建和管理窗口
- 访问文件系统
- 处理系统托盘
- 提供 API 给渲染进程调用

**关键文件：**
- [index.ts](../packages/electron/src/index.ts) - 主进程入口
- [window/](../packages/electron/src/window/) - 窗口管理
- [ipc-server/](../packages/electron/src/ipc-server/) - 进程通信
- [dataSource/](../packages/electron/src/dataSource/) - 数据源管理

#### packages/renderer - 渲染进程

**作用：** 用户界面，显示和交互

**类比：** 像餐厅的前台

**主要功能：**
- 显示界面（基于 React）
- 处理用户输入
- 显示编辑器和预览器
- 管理应用状态

**关键文件：**
- [main/index.tsx](../packages/renderer/src/main/index.tsx) - 渲染进程入口
- [main/App.tsx](../packages/renderer/src/main/App.tsx) - 应用根组件
- [main/stores/](../packages/renderer/src/main/stores/) - 状态管理
- [components/](../packages/renderer/src/components/) - UI 组件

#### packages/shared - 共享代码

**作用：** 在主进程和渲染进程间共享

**主要内容：**
- TypeScript 类型定义
- 通用工具函数
- 常量定义

## 第三课：理解核心概念

### 3.1 Electron 主进程与渲染进程

#### 主进程（Main Process）

**特点：**
- 每个 Electron 应用只有一个主进程
- 运行 Node.js 环境
- 可以访问系统资源
- 不能直接操作 DOM

**职责：**
- 创建应用窗口
- 管理应用生命周期
- 处理系统级操作

```typescript
// packages/electron/src/index.ts
app.whenReady()
  .then(() => restoreOrCreateWindow('main')) // 创建窗口
  .then(() => pluginManager.loadAll());      // 加载插件
```

#### 渲染进程（Renderer Process）

**特点：**
- 每个窗口都有一个渲染进程
- 运行 Chromium 环境
- 可以操作 DOM
- 默认不能访问 Node.js API（安全考虑）

**职责：**
- 渲染 UI
- 处理用户交互
- 显示页面内容

```typescript
// packages/renderer/src/main/index.tsx
root.render(
  <FluentProvider theme={webLightTheme}>
    <App />
  </FluentProvider>,
);
```

### 3.2 IPC 通信

主进程和渲染进程是隔离的，需要通过 IPC（进程间通信）交换数据。

#### 通信流程

```
渲染进程                    主进程
    │                         │
    │  invoke('method')       │
    ├───────────────────────►│
    │                         │  Handler处理
    │                         │
    │  ◄──────────────────────┤
    │      return result      │
```

#### 实际例子

**主进程定义 Handler：**
```typescript
// packages/electron/src/ipc-server/handlers/DataSourceHandler.ts
class DataSourceHandler {
  async list(event, path) {
    return await dataSource.list(path);
  }
}
```

**渲染进程调用：**
```typescript
// packages/renderer/src/main/services/file-service.ts
const files = await window.api.invoke('DataSource.list', '/path');
```

### 3.3 MobX 状态管理

ONote 使用 MobX 管理应用状态。

#### 什么是状态？

状态 = 应用中的数据

例如：
- 当前打开的文件
- 用户设置
- 文件列表
- UI 布局状态

#### Store 示例

```typescript
// packages/renderer/src/main/stores/SettingStore.ts
class SettingStore {
  theme = 'light';
  fontSize = 14;

  constructor() {
    makeAutoObservable(this);
  }

  setTheme(theme) {
    this.theme = theme;
  }
}
```

#### 在组件中使用

```typescript
import { observer } from 'mobx-react-lite';

const ThemedComponent = observer(() => {
  const { settingStore } = stores;

  return (
    <div style={{ color: settingStore.theme === 'dark' ? 'white' : 'black' }}>
      Hello
    </div>
  );
});
```

**工作原理：**
1. 组件通过 `observer` 包装
2. 组件访问 Store 中的状态
3. 状态改变时，组件自动重新渲染

### 3.4 React 组件架构

ONote 的 UI 组件层级：

```
App (应用根组件)
├── Sidebar (侧边栏 - 文件树)
├── Page (主页面)
│   ├── Editor (Monaco 编辑器)
│   └── Previewer (Markdown 预览)
└── EventBus (事件总线)
```

## 第四课：添加第一个功能

### 4.1 功能需求

我们将添加一个简单的功能：**显示当前时间的组件**

**目标：**
- 在侧边栏底部显示当前时间
- 每秒更新一次
- 学习组件开发流程

### 4.2 创建组件

#### 步骤 1：创建组件文件

```bash
# 在 packages/renderer/src/components/ 下创建
touch packages/renderer/src/components/CurrentTime.tsx
touch packages/renderer/src/components/CurrentTime.module.scss
```

#### 步骤 2：实现组件

编辑 `CurrentTime.tsx`：

```typescript
import React, { FC, useEffect, useState } from 'react';
import styles from './CurrentTime.module.scss';

const CurrentTime: FC = () => {
  const [time, setTime] = useState(new Date());

  useEffect(() => {
    // 每秒更新时间
    const timer = setInterval(() => {
      setTime(new Date());
    }, 1000);

    // 组件卸载时清除定时器
    return () => clearInterval(timer);
  }, []);

  return (
    <div className={styles.container}>
      <span className={styles.label}>当前时间：</span>
      <span className={styles.time}>{time.toLocaleTimeString()}</span>
    </div>
  );
};

export default CurrentTime;
```

#### 步骤 3：添加样式

编辑 `CurrentTime.module.scss`：

```scss
.container {

  font-size: 12px;
  color: #666;
  display: flex;
  gap: 5px;
}

.label {
  font-weight: bold;
}

.time {
  font-family: monospace;
}
```

### 4.3 集成到应用

### 4.5 添加交互

让我们增加一个功能：点击时间可以切换格式

```typescript
const CurrentTime: FC = () => {
  const [time, setTime] = useState(new Date());
  const [is24Hour, setIs24Hour] = useState(true);

  // ... 现有的 useEffect 代码

  const formatTime = (date: Date) => {
    return is24Hour
      ? date.toLocaleTimeString('zh-CN', { hour12: false })
      : date.toLocaleTimeString('zh-CN', { hour12: true });
  };

  return (
    <div
      className={styles.container}
      onClick={() => setIs24Hour(!is24Hour)}
      style={{ cursor: 'pointer' }}
    >
      <span className={styles.label}>当前时间：</span>
      <span className={styles.time}>{formatTime(time)}</span>
      <span className={styles.hint}>({is24Hour ? '24小时' : '12小时'})</span>
    </div>
  );
};
```

**排查步骤：**
1. 检查命名空间和方法名拼写

### 5.3 实践练习

**任务：** 调试一个有问题的组件

这个组件有 bug，尝试找出并修复：

```typescript
// buggy-component.tsx
import React, { FC } from 'react';
import { observer } from 'mobx-react-lite';
import stores from '/@/main/stores';

const BuggyComponent: FC = observer(() => {
  const [count, setCount] = stores.counterStore.count;

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => stores.counterStore.increment()}>
        Increment
      </button>
    </div>
  );
});

```

**问题提示：**
1. `useState` 的使用是否正确？
2. MobX store 的数据如何访问？
3. 点击按钮后状态会更新吗？

**正确答案：**
```typescript
import React, { FC } from 'react';
import { observer } from 'mobx-react-lite';
import stores from '/@/main/stores';

const FixedComponent: FC = observer(() => {
  // ❌ 错误：不需要 useState
  // const [count, setCount] = stores.counterStore.count;

  // ✅ 正确：直接访问 store
  const { count } = stores.counterStore;

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => stores.counterStore.increment()}>
        Increment
      </button>
    </div>
  );
});

export default FixedComponent;
```

### 6.1 为什么需要测试？

### 6.2 测试类型

#### 单元测试

测试单个函数或组件

```typescript
// your-component.test.ts
import { render, screen } from '@testing-library/react';
import YourComponent from './YourComponent';

describe('YourComponent', () => {
  it('should render hello text', () => {
    render(<YourComponent />);
    const element = screen.getByText('Hello');
    expect(element).toBeInTheDocument();
  });
});
```

#### 集成测试

测试多个模块的交互

```typescript
// integration.test.ts
describe('File Open Flow', () => {
  it('should open file and display content', async () => {
    // 1. 点击文件
    await userEvent.click(fileItem);

    // 2. 等待加载

      expect(editor.getContent()).toBe('file content');
    });
  });

```

### 6.3 编写第一个测试

我们为之前创建的 `CurrentTime` 组件编写测试：

```typescript
// packages/renderer/src/components/CurrentTime.test.tsx
import { render, screen } from '@testing-library/react';
import { act } from 'react-dom/test-utils';
import CurrentTime from './CurrentTime';

describe('CurrentTime', () => {
  beforeEach(() => {
    jest.useFakeTimers();
  });

  afterEach(() => {
    jest.useRealTimers();
  });

  it('should render current time', () => {
    render(<CurrentTime />);

    expect(screen.getByText(/当前时间/)).toBeInTheDocument();
  });

  it('should update time every second', () => {
    render(<CurrentTime />);

    const initialTime = screen.getByText(/\d{2}:\d{2}:\d{2}/);

    // 快进 1 秒
    act(() => {
      jest.advanceTimersByTime(1000);
    });

    const updatedTime = screen.getByText(/\d{2}:\d{2}:\d{2}/);
    expect(updatedTime).not.toBe(initialTime);
  });

  it('should toggle format on click', async () => {
    render(<CurrentTime />);

    const timeElement = screen.getByText(/\d{2}:\d{2}:\d{2}/);

    // 点击切换格式
    await userEvent.click(timeElement);

    // 验证格式改变
    expect(screen.getByText(/\(24小时\)/|\(12小时\)/)).toBeInTheDocument();
  });
});
```

### 6.5 测试最佳实践

#### 1. 测试命名

```typescript
// ✅ 好的命名
it('should increment counter when button clicked')
it('should display error message when API fails')

// ❌ 不好的命名
it('test counter')
it('works')
```

#### 2. AAA 模式

```typescript
it('should add item to cart', () => {
  // Arrange (准备)
  const cart = new Cart();
  const item = { id: 1, name: 'Product' };

  // Act (执行)
  cart.addItem(item);

  // Assert (断言)
  expect(cart.items).toContain(item);
});
```

#### 3. 避免测试实现细节

```typescript
// ❌ 测试实现细节
it('should set state variable to true')
expect(component.state.isVisible).toBe(true);

// ✅ 测试行为
it('should show modal when button clicked')
expect(screen.getByRole('dialog')).toBeVisible();
```
